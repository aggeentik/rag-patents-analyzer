# Hybrid Retrieval and LLM Integration Guide

This guide covers the implementation and usage of Phase 3b (Hybrid Retrieval with RRF) and Phase 4 (LLM Integration) of the Patents Analyzer system.

## Overview

The system now implements a complete RAG (Retrieval-Augmented Generation) pipeline:

1. **Hybrid Retrieval** - Combines three retrieval methods using RRF (Reciprocal Rank Fusion):
   - **BM25** - Sparse keyword matching (good for exact terms like chemical symbols)
   - **Semantic (FAISS)** - Dense vector search (good for conceptual similarity)
   - **Graph** - Knowledge graph traversal (good for entity relationships)

2. **Cross-Encoder Reranking** (optional) - Post-fusion reranking using `BAAI/bge-reranker-v2-m3`

3. **LLM Integration** - Uses LiteLLM for unified access to:
   - **Ollama** - Local models (llama3.1, mistral, etc.)
   - **AWS Bedrock** - Claude, Titan, Llama models
   - **Azure AI** - GPT-4.1, etc.
   - **OpenAI** - GPT models

## Architecture

```
User Query
    ↓
[Entity Extraction] ← Identifies entities in query
    ↓
[Hybrid Retriever]
    ├── BM25 Retriever (keyword matching)
    ├── Semantic Retriever (vector similarity)
    └── Graph Retriever (entity graph traversal)
    ↓
[RRF Fusion] ← Combines results using Reciprocal Rank Fusion
    ↓
[Reranker] ← Optional cross-encoder reranking (BAAI/bge-reranker-v2-m3)
    ↓
[Answer Generator]
    ├── Context Building (top K chunks)
    ├── Prompt Construction
    └── LLM Generation (Ollama/Bedrock)
    ↓
Structured Answer + Sources
```

### Programmatic Usage

#### 1. Initialize Retrievers

```python
import json
from pathlib import Path
from src.retrieval import BM25Retriever, SemanticRetriever, GraphRetriever, HybridRetriever
from src.knowledge_graph.store import KnowledgeGraphStore

# Load chunks
with open("data/processed/patents.json") as f:
    data = json.load(f)
chunks = data["chunks"]

# BM25 Retriever
bm25 = BM25Retriever.load("data/processed/bm25_index.pkl", chunks)

# Semantic Retriever
semantic = SemanticRetriever.load(
    "data/processed/faiss.index",
    "data/processed/chunk_ids.json",
    chunks
)

# Graph Retriever
kg_store = KnowledgeGraphStore("data/processed/knowledge_graph.db")
kg_store.connect()
graph = GraphRetriever.load(
    path="",
    chunks=chunks,
    kg_store=kg_store,
    max_hops=2,
    score_decay=0.5
)

# Hybrid Retriever
hybrid = HybridRetriever(
    bm25_retriever=bm25,
    semantic_retriever=semantic,
    graph_retriever=graph,
    weights={"bm25": 1.0, "semantic": 1.0, "graph": 0.5},
    rrf_k=60
)
```

#### 2. Retrieve Documents

```python
query = "What is the effect of silicon on magnetic properties?"

# Retrieve top 10 chunks
results = hybrid.search(query, top_k=10)

# Each result has:
# - rrf_score: Combined RRF score
# - bm25_rank, semantic_rank, graph_rank: Individual ranks
# - score_breakdown: Human-readable score explanation
# - content: Chunk text
# - metadata: patent_id, section, page, etc.

for i, result in enumerate(results, 1):
    print(f"[{i}] RRF: {result['rrf_score']:.4f}")
    print(f"    {result['score_breakdown']}")
    print(f"    {result['content'][:100]}...")
```

#### 3. Generate Answers

```python
from src.llm import LLMClient, AnswerGenerator

# Initialize LLM
llm = LLMClient.from_env()  # Loads from .env
# OR explicitly:
# llm = LLMClient(model="ollama/llama2")
# llm = LLMClient(model="bedrock/anthropic.claude-3-sonnet-20240229-v1:0")

# Create answer generator
generator = AnswerGenerator(
    llm_client=llm,
    max_context_chunks=5,
    include_metadata=True
)

# Generate answer
result = generator.generate_answer(
    question=query,
    retrieved_chunks=results,
    temperature=0.0,
    stream=True  # Stream output to console
)

print(result["answer"])
print(f"\nSources: {len(result['sources'])}")
for source in result["sources"]:
    print(f"  - {source['patent_id']} (RRF: {source['rrf_score']:.4f})")
```

## How It Works

### RRF (Reciprocal Rank Fusion)

RRF combines rankings from multiple retrievers using the formula:

```
RRF_score(chunk) = Σ weight_i / (k + rank_i)
```

Where:
- `k` = constant (typically 60) to reduce impact of high ranks
- `rank_i` = rank from retriever i (1, 2, 3, ...)
- `weight_i` = weight for retriever i

**Example:**
```
Chunk X:
  BM25 rank: 2 → weight=1.0 → 1.0 / (60 + 2) = 0.0161
  Semantic rank: 5 → weight=1.0 → 1.0 / (60 + 5) = 0.0154
  Graph rank: 1 → weight=0.5 → 0.5 / (60 + 1) = 0.0082
  RRF score = 0.0161 + 0.0154 + 0.0082 = 0.0397
```

### Graph Retriever Algorithm

1. **Extract entities from query** using EntityExtractor
2. **Find matching entities in knowledge graph**
3. **BFS traversal** from each entity (up to `max_hops`)
4. **Score chunks**:
   - Direct match: score = 1.0
   - 1-hop away: score = 0.5 (with `score_decay=0.5`)
   - 2-hop away: score = 0.25
5. **Aggregate scores** for chunks containing multiple entities

### LLM Prompt Structure

The Answer Generator constructs prompts with:

```
SYSTEM: You are a patent analysis assistant...

USER:
Based on the following patent excerpts, answer the question.

PATENT EXCERPTS:
[Source 1] (Patent: X, Section: Y, Page: Z)
<chunk content>

[Source 2] ...

QUESTION:
<user question>

INSTRUCTIONS:
1. Answer based ONLY on excerpts
2. Include technical details, numbers, formulas
3. Reference sources [Source X]
4. State if information is missing
5. Use technical language
```

## Configuration

### Retriever Weights

Adjust based on your use case:

```python
# Balanced (default)
weights = {"bm25": 1.0, "semantic": 1.0, "graph": 0.5}

# Keyword-focused (exact terms, chemical symbols)
weights = {"bm25": 2.0, "semantic": 0.5, "graph": 0.3}

# Semantic-focused (conceptual questions)
weights = {"bm25": 0.5, "semantic": 2.0, "graph": 0.8}

# Graph-focused (entity relationships)
weights = {"bm25": 0.5, "semantic": 0.5, "graph": 1.5}
```

### Graph Traversal

```python
graph = GraphRetriever(
    kg_store=kg_store,
    max_hops=2,      # How far to traverse (1, 2, or 3)
    score_decay=0.5  # How quickly scores decrease (0.0-1.0)
)
```

### Cross-Encoder Reranking (Optional)

Enable via environment variables:

```env
RERANKER_ENABLED=true
RERANKER_MODEL=BAAI/bge-reranker-v2-m3
```

Or programmatically:

```python
from src.retrieval import CrossEncoderReranker, reranker_from_env

# From environment
reranker = reranker_from_env()  # Returns None if RERANKER_ENABLED != true

# Or explicitly
reranker = CrossEncoderReranker(model_name="BAAI/bge-reranker-v2-m3")
scores = reranker.score(query, results)
```

The reranker jointly encodes (query, document) pairs using a cross-encoder model for more accurate relevance scoring. It is applied after RRF fusion to the top candidate pool.

## Performance Tuning

### For Speed

```python
# Use fewer context chunks
generator = AnswerGenerator(max_context_chunks=3)

# Retrieve fewer candidates
results = hybrid.search(query, top_k=5, retriever_top_k=15)

# Use faster LLM

```

### For Quality

```python
# Use more context
generator = AnswerGenerator(max_context_chunks=10)

# Retrieve more candidates for fusion
results = hybrid.search(query, top_k=10, retriever_top_k=50)

# Use better LLM
llm = LLMClient(model="bedrock/anthropic.claude-3-sonnet-20240229-v1:0")
```

## Troubleshooting

### "Cannot connect to Ollama"

**Solution:**
```bash
# Start Ollama server
ollama serve

# In another terminal, pull a model
ollama pull llama2
```

### "Missing AWS credentials"

**Solution:**
Set in `.env`:
```env
AWS_ACCESS_KEY_ID=your_key
AWS_SECRET_ACCESS_KEY=your_secret
AWS_REGION_NAME=us-east-1
```

Or use AWS CLI:
```bash
aws configure
```

### "No entities found in query"

The graph retriever needs entities to work. If your query has no entities:
- The graph retriever returns empty results
- BM25 and semantic retrievers still work
- RRF fusion handles missing retrievers gracefully

### Low-quality answers

Try:
1. Increase `max_context_chunks` for more context
2. Use a better LLM model (Claude Sonnet instead of Llama2)
3. Adjust retriever weights to emphasize better-performing retrievers
4. Increase `retriever_top_k` for better RRF fusion

### Out of memory

Try:
1. Reduce `max_context_chunks`
2. Reduce `retriever_top_k`
3. Use a smaller LLM model
4. Process fewer patents initially
